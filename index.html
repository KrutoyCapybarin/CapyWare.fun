CapyWare

External FFlag executor for Roblox. Writes client-side feature flags directly to memory — no DLL injection, no hooks. Scripting engine included.

↓ Download GitHub →

Installation

Первый запуск — автоматический установщик. Просто скачай и запусти.

Первый запуск (с Desktop или Downloads)

При запуске CapyWare.exe с рабочего стола или папки загрузок:

  1. Создаётся папка %DESKTOP%/CapyWare/ с подпапками
  2. Exe копируется в папку
  3. Создаётся ярлык CapyWare.lnk на рабочем столе
  4. Запускается установленная копия
  5. Оригинальный файл удаляется
ℹ
После установки используй ярлык на рабочем столе или exe из папки CapyWare.

Зависимости (устанавливаются автоматически)

PyQt5     — UI
keyboard  — глобальные хоткеи
requests  — HTTP для request()

Folder Structure

CapyWare/ ├── autoexec/ ← скрипты запускаются при каждом Inject ├── scripts/ ← библиотека скриптов (Open/Save) ├── workspace/ ← рабочая папка для readfile/writefile ├── CapyWare.exe └── config.cfg ← настройки (JSON, сохраняются автоматически)
ℹ
autoexec/ — все файлы .cw .ff .txt запускаются в алфавитном порядке при нажатии ⚡ Inject.

FFlags ★

Основная функция CapyWare — запись FFlag значений в память Roblox. Два формата синтаксиса.

Key = Value

-- числовой флаг
TaskSchedulerTargetFps = 9999

-- булев флаг
DebugDisplayFPS = true
DisableDPIScale = false

-- значения true/false/1/0 — все работают
DebugDisplayFPS = 1
ℹ
Заглавная буква в начале имени = FFlag. Строчная буква = переменная скрипта.

JSON Mode

Если текст начинается с { — автоматически парсится как JSON. Приоритет над скриптовым движком.

{
  "TaskSchedulerTargetFps": 9999,
  "DebugDisplayFPS": true,
  "DisableDPIScale": true
}
ℹ
JSON режим можно выключить в Settings → Auto JSON mode.

Prefix Stripping

Если флаг не найден в дампе — CapyWare отрезает известный префикс и ищет bare-имя. Работает по всем ring'ам.

Что написалЧто ищется
DFIntTaskSchedulerTargetFps→ TaskSchedulerTargetFps
FFlagDebugDisplayFPS→ DebugDisplayFPS
DFLogAbuseReport→ AbuseReport

Поддерживаемые префиксы: DFInt DFFlag DFLog DFString FInt FFlag FLog FString SFInt SFFlag SFLog SFString

Dump System

Флаги ищутся по кольцам приоритета. Ring 0 всегда первый.

RING 0Основной дамп (default: GitHub current_dump.h). Ищется первым.
RING 1Резервный. Используется если флаг не найден в ring 0.
RING 2Второй резервный. Количество рингов не ограничено.
-- добавить дамп (формат: auto / json / h)
adddump("https://example.com/dump.h", "backup", "auto")

-- переставить дамп в нужный ринг
usedump("backup", 1)

-- посмотреть все загруженные дампы
getdumps()
Формат дампаПример
C++ header (.h)inline constexpr FFlag Foo = 0x1234abc;
JSON{"FlagName": "0x1234"} или {"FlagName": 1234}

Variables

Переменные скрипта — строчная буква в начале имени. FFlag — заглавная.

-- локальная переменная (только в текущем скрипте)
local x = 25
local name = "capy"
local flag = true

-- присваивание без local — тоже локальная
count = 0

-- конкатенация строк
local msg = "hello " .. name
print(msg)   -- hello capy

-- это FFlag, не переменная (заглавная буква)
TaskSchedulerTargetFps = 9999

Control Flow

if / elseif / else / end

local x = 10

-- однострочный
if x == 10 then print("ten") end

-- многострочный
if x == 10 then
    TaskSchedulerTargetFps = 9999
    print("fps unlocked")
elseif x == 5 then
    print("five")
else
    print("other")
end

while / do / end + break / continue

local i = 0
while i < 5 do
    i = i + 1
    if i == 3 then continue end  -- пропустить итерацию
    if i == 5 then break end     -- выйти из цикла
    print(i)
end

wait()

TaskSchedulerTargetFps = 9999
wait(2)        -- пауза 2 секунды (блокирует скрипт)
print("done")
wait()         -- без аргумента = 0 секунд

Functions

Пользовательские функции сохраняются в памяти до закрытия CapyWare и доступны из любого скрипта.

-- объявление
function boost()
    TaskSchedulerTargetFps = 9999
    DebugDisplayFPS = true
    print("boosted!")
end

-- вызов
boost()

-- функции доступны из других скриптов пока капивар открыт
boost()
⚠
Функции без аргументов. Параметры не поддерживаются — используй глобальные переменные для передачи данных.

Events

Ивенты регистрируют хендлеры которые срабатывают автоматически. Живут до ⏏ Eject.

OnKeyPress

OnKeyPress("Q"):
    TaskSchedulerTargetFps = 9999
    print("Q — fps unlocked")
end

OnKeyPress("E"):
    TaskSchedulerTargetFps = 60
end
ℹ
Хоткей срабатывает глобально — даже когда CapyWare свёрнут. Все хендлеры очищаются при Eject.

OnAttach

-- срабатывает когда ⚡ Inject успешен
OnAttach():
    print("attached!")
    TaskSchedulerTargetFps = 9999
end

OnEject

-- срабатывает когда нажата ⏏ Eject
OnEject():
    print("bye")
end

Global Variables

Глобальные переменные живут пока CapyWare открыт и доступны из любого скрипта, функции и autoexec.

-- объявить глобальную переменную
global myfps = 9999
global username = "capy"

-- в другом скрипте — переменная уже есть
print(myfps)    -- 9999

-- посмотреть все глобальные
list_globals()

-- очистить все глобальные
capyware.clear_globals()

Filesystem

Все пути относительны папке CapyWare. Требует Executor Mode в Settings.

readfile("path")
Вернуть содержимое файла как строку. Выведет в консоль.
writefile("path", "data")
Записать данные в файл. Создаст если нет, перезапишет если есть.
appendfile("path", "data")
Дописать в конец файла.
delfile("path")
Удалить файл.
isfile("path")
Проверить существование файла. true/false в консоль.
isfolder("path")
Проверить существование папки.
makefolder("path")
Создать папку (и родительские если нужно).
delfolder("path")
Удалить папку со всем содержимым.
listfiles("path")
Список файлов в папке. Выведет в консоль.
loadfile("path")
Прочитать и выполнить скрипт из файла.
writefile("workspace/log.txt", "session started")
appendfile("workspace/log.txt", "\nfps = 9999")
readfile("workspace/log.txt")
loadfile("scripts/boost.cw")

HTTP запросы из скрипта. Discord вебхуки, внешние API и т.д.

-- простой GET
request({"url": "https://httpbin.org/get", "method": "GET"})

-- Discord webhook
request({
    "url": "https://discord.com/api/webhooks/ID/TOKEN",
    "method": "POST",
    "headers": {"Content-Type": "application/json"},
    "body": "{\"content\": \"injected\"}"
})

capyware.* API

capyware.offsets()
Перезагрузить ring 0 дамп с GitHub.
capyware.eject()
Eject — освободить хендл, очистить хоткеи. Аналог кнопки ⏏.
capyware.undoall()
Откатить все изменённые флаги к исходным значениям (undo log).
capyware.notify("msg")
Вывести сообщение в консоль с префиксом 🔔.
capyware.attached()
Вывести true/false — подключён ли к Roblox.
capyware.version()
Вывести текущую версию CapyWare.
capyware.currentfps()
Прочитать текущее значение TaskSchedulerTargetFps из памяти Roblox.
capyware.clear_globals()
Очистить все глобальные переменные.
capyware.exec("python code")
Выполнить произвольный Python код. Код в кавычках.
-- пример: при инжекте обновить дамп и залогировать
OnAttach():
    capyware.offsets()
    capyware.notify("attached!")
    capyware.currentfps()
end

-- выполнить python
capyware.exec("import os; os.startfile('notepad.exe')")
